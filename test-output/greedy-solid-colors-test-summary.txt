ROOT CAUSE ANALYSIS:
===================

The greedy algorithm appears to have a fundamental issue with solid color
regions, particularly white. Instead of converging to a uniform byte pattern
(e.g., 0xFF for white), it alternates between multiple patterns, likely due to:

1. **Smoothness Penalty Overreach**: The SMOOTHNESS_PENALTY (1,000,000) is
   designed to prevent vertical stripes, but it may be forcing excessive
   variation even in solid color regions where uniformity is correct.

2. **History-Based Penalties**: The algorithm tracks last 5 scanlines and
   penalizes byte repetition. For solid colors, repetition IS the correct
   behavior, not something to penalize.

3. **NTSC Rendering Artifact**: The alternating byte patterns (0xFF, 0x7F,
   0xFE, 0x7E) may be creating color artifacts through NTSC phase interactions
   that render as non-white colors.

RECOMMENDATIONS:
================

1. Detect solid color regions and disable smoothness penalties in those areas
2. Reduce or eliminate vertical history penalties for uniform input colors
3. Add solid color detection: when target colors are identical for N
   consecutive bytes, use uniform byte pattern instead of variation
4. Consider separate code path for solid colors that bypasses error diffusion
